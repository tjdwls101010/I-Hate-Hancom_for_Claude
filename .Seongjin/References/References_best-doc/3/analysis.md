## II. 암묵지 (경험자만 아는 디자인 감각)

### 1. 굵은 강조(Bold)의 패턴: "핵심 가치어 강조" 원칙

이 문서는 본문에서 굵은 강조를 **매우 적극적으로, 그러나 규칙적으로** 사용한다. 단순히 "중요한 단어"를 굵게 하는 것이 아니라, 독자(기자/국민)가 **문장을 빠르게 스캔했을 때 정책의 핵심이 즉시 드러나도록** 설계되어 있다.

#### 패턴 A: "정책 내용 + 구체적 일시/수치"를 한 쌍으로 강조

```
"예비창업자와 [BOLD]창업기업[/BOLD]의 [BOLD]고민[/BOLD]을 [BOLD]한 곳[/BOLD]에서
[BOLD]상담[/BOLD]받을 수 있는 [BOLD]'온라인 스타트업 원스톱 지원센터'를 3월 30일(월) 개소[/BOLD]한다고 밝혔다."
```

- "예비창업자와"는 일반체 (charPr[40]: 14pt 바탕)
- "창업기업", "고민", "한 곳", "상담"은 각각 별도 run으로 BOLD (charPr[47]: 14pt 바탕 BOLD)
- "'온라인 스타트업 원스톱 지원센터'를 3월 30일(월) 개소"는 하나의 BOLD run

**규칙**: 핵심 정책명 + 핵심 일자가 하나의 BOLD 구간에 함께 들어간다.

#### 패턴 B: "성과 수치"의 선택적 강조

```
"[BOLD]지난 4개월간[/BOLD] 법률, 세무... [BOLD]약 7,600여 건[/BOLD]의 애로 사항이 접수...
이 중 [BOLD]93%를 상담 당일 즉시 해결[/BOLD]함으로써"
```

- "지난 4개월간" = charPr[54] BOLD: 기간
- "약 7,600여 건" = charPr[54] BOLD: 핵심 수치
- "93%를 상담 당일 즉시 해결" = charPr[103] BOLD: 성과 + 수치 조합
- "법률, 세무, 경영, 마케팅 등 다양한 분야에서" = charPr[55] 일반: 열거는 강조하지 않음

**규칙**: 숫자 자체가 아니라 "숫자 + 그 숫자가 의미하는 성과"를 묶어서 강조한다.

#### 패턴 C: "키워드 군집" 강조 (정책 방향 서술시)

```
"[BOLD]현장 경험이 풍부한 대기업 퇴직자[/BOLD] 및 [BOLD]선배 창업가[/BOLD]들의
[BOLD]경영 전략, 투자 준비[/BOLD], [BOLD]사업 운영 전반의 실질적인 멘토링 사례[/BOLD] 등
[BOLD]민간 중심 지원 체계의 확산 가능성[/BOLD]도 확인했다."
```

- "들의", "등", "도 확인했다" = 일반체 (조사/서술어)
- 주어, 목적어, 핵심 명사구 = BOLD

**규칙**: 조사와 어미는 일반체, 명사구/핵심 서술은 BOLD. 한 문장 안에서 BOLD와 일반이 5~7회 전환된다.

#### 패턴 D: 장관 인용문에서의 강조

```
"[BOLD]원스톱 지원센터[/BOLD]는 창업자가 여러 기관을 찾아다니지 않아도 되는
[BOLD]정책 접근 방식[/BOLD]의 [BOLD]전환[/BOLD]"이라며"
```

- 정책명은 항상 BOLD
- 핵심 가치어("전환", "확대", "기반", "마련") 역시 BOLD
- 접속사, 조사는 일반

### 2. 소제목 인용 박스: 섹션 전환의 신호

이 문서는 본문을 3개의 논리적 섹션으로 나누며, 각 전환점에 **따옴표+밑줄+굵게** 소제목을 배치한다:

1. `"원스톱 상담 수요 확인 … 4개월간 상담 7천여 건"` -- 성과 보고 섹션
2. `"온라인 서비스 도입 … 언제 어디서나 창업 고민 상담"` -- 신규 서비스 소개 섹션
3. `"모두의 창업 프로젝트와 연계 … 창업 접근성 확대"` -- 향후 계획 섹션

**XML 구현**:
```xml
<hp:run charPrIDRef="61"><hp:t>"원스톱 상담 수요 확인</hp:t></hp:run>
<hp:run charPrIDRef="66"><hp:t> … </hp:t></hp:run>
<hp:run charPrIDRef="62"><hp:t>4개월간 상담 7천여 건</hp:t></hp:run>
<hp:run charPrIDRef="61"><hp:t>"</hp:t></hp:run>
```

- charPr[61]: 14pt 한컴 윤고딕 230 **BOLD + underline(BOTTOM, SOLID, #000000)**
- charPr[66]: 14pt 한컴 윤고딕 230 **BOLD** (밑줄 없음) -- `…` 부분만 밑줄 제거
- charPr[62]: 14pt 한컴 윤고딕 230 **BOLD + underline(BOTTOM, SOLID, #000000)**

**암묵 규칙**: `…` (말줄임표)에는 밑줄을 적용하지 않는다. 시각적으로 밑줄과 점이 겹쳐 지저분해지기 때문. 이것은 경험 있는 작성자만 아는 미세 조정이다.

**소제목 패턴**: "핵심 키워드 … 보완 키워드" 형태로, 앞부분은 현황/문제, 뒷부분은 결과/방향.

### 3. 표와 강조 박스의 구분: 테두리 스타일이 곧 의미

이 문서에는 3가지 유형의 박스가 있고, 테두리 스타일로 구분된다:

| 유형 | 테두리 | 배경 | 용도 |
|------|---------|------|------|
| **개요 박스** | CIRCLE (둥근 점선) 0.3mm 외곽 + DASH (점선) 0.12mm 내부 | 없음 | 정책 개요 요약 |
| **사례 박스** | SOLID 0.2mm 외곽 + 장식선 (상단 좌우 가로선) | 없음 | 상담 사례 나열 |
| **데이터 표** | SOLID 0.12mm 균일 | 헤더 #DFE6F7 | 정량적 데이터 정리 |

**암묵 규칙**:
- **둥근 점선 박스** = "이건 보조 정보입니다. 본문 흐름과 별개로 참고하세요" 신호
- **장식선 사례 박스** = "구체적 스토리텔링 영역" 신호
- **실선 표** = "정형 데이터" 신호
- 배경색은 표 헤더에만 사용 (#DFE6F7 연파랑). 본문 강조 박스에는 배경색을 쓰지 않는다.

### 4. 마커 위계 체계

이 문서에서 사용하는 마커의 위계:

```
Level 0 (소제목):  "따옴표 + 밑줄 + 굵게"  -- 섹션 전환
Level 1 (본문):    (마커 없음)              -- 일반 본문 단락
Level 1.5 (꺾쇠):  < 스타트업 원스톱 지원센터 개요 >  -- 부속 정보 제목
Level 2 (박스 내): ‣ (추진 배경) / ‣ (지원 내용)     -- 개요 박스 내 항목
Level 2.5 (번호):  ➊ ➋ ➌                  -- 지원 내용 세부 나열
Level 3 (열거):    ▲                       -- 본문 내 병렬 나열
Level 3 (사례):    ▪ (사례1) (사례2) (사례3) -- 사례 박스 내 항목
Level 4 (각주):    *                        -- 각주/주석
```

**암묵 규칙**:
- `‣` (삼각형 불릿)은 개요 박스 안에서만 쓴다. 본문에서는 절대 안 씀
- `▲`은 본문 내 병렬 나열에만 쓴다 (예: "▲창업패키지 등 사업화 지원사업 ▲정책자금...")
- `▪` (작은 사각형)은 사례 박스에서만 쓴다
- `➊➋➌`은 `‣` 항목 내부의 세부 나열에만 쓴다
- 일반 본문에서는 어떤 마커도 쓰지 않는다. 들여쓰기와 Bold로 구조를 만든다

### 5. 숫자 강조의 미학

숫자 자체가 아니라 "숫자 + 맥락"을 강조하는 패턴:

| 원문 | 강조 범위 | 강조 방식 |
|------|-----------|----------|
| "약 7,600여 건" | "약 7,600여 건" 전체 | BOLD (charPr[54]) |
| "93%를 상담 당일 즉시 해결" | "93%를 상담 당일 즉시 해결" 전체 | BOLD (charPr[103]) |
| "10점 만점 중 9.7점" | "10점 만점 중 9.7점" 전체 | BOLD (charPr[47]) |
| "2,000여명 전문가" | "2,000여명 전문가" 전체 | BOLD (charPr[71]) |
| "평균 3일 내외로 진행" | "평균 3일 내외로 진행" 전체 | BOLD (charPr[101]) |
| "기업당 연간 300만원 한도" | "기업당 연간 300만원 한도" 전체 | BOLD (charPr[48]) |

**암묵 규칙**:
- 숫자만 따로 굵게 하지 않는다. "숫자 + 단위 + 의미"를 하나의 BOLD 구간으로 묶는다
- "86.6%"(각주 내)는 강조하지 않는다 -- 각주는 보조 정보이므로 강조 대상이 아님
- 핵심 성과 숫자(7,600건, 93%, 9.7점)만 강조하고, 부수적 숫자(17개, 12월 1일)는 맥락에 따라 결정

### 6. 들여쓰기와 공백 리듬

**본문 단락 시작**: 2칸 들여쓰기 (실제 XML에서 `hp:t` 시작에 전각 스페이스 2개)
```xml
<hp:run charPrIDRef="40"><hp:t>  중소벤처기업부(장관 한성숙, ...</hp:t></hp:run>
```

**단락 간 공백**: 빈 단락 1개 삽입 (빈 `hp:run`으로 구현)
```xml
<hp:p id="0" paraPrIDRef="41" styleIDRef="0" ...><hp:run charPrIDRef="40"/></hp:p>
```

**각주(*)의 들여쓰기**: 본문과 동일한 2칸 들여쓰기 + 12pt 맑은 고딕
```
  * 오프라인 스타트업 원스톱 지원센터 구축('25.12.1) → 온라인 센터 개소('26.3.30)
```

**소제목 전후 공백**: 소제목 앞뒤로 각각 빈 단락 1개씩 = 시각적 호흡

### 7. 하이퍼링크 처리

- 일반 URL: charPr[46] = 12pt 맑은 고딕 **밑줄(#0000FF 파란색)** -- 각주 내 URL
- 본문 URL: charPr[135] = 14pt 바탕 **BOLD + 밑줄(#800080 보라색)** -- 본문 내 강조 URL
- `hp:fieldBegin type="HYPERLINK"` + `hp:fieldEnd`로 감싸서 클릭 가능하게 처리

### 8. 폰트 사용의 위계

```
HY헤드라인M (17pt/16pt)      -- 참고 페이지 제목 (가장 높은 위계)
함초롬돋움 (22pt BOLD)        -- 메인 제목
한컴 윤고딕 230 (14pt BOLD)   -- 소제목 인용 (밑줄 동반)
바탕 (14pt)                  -- 본문 (BOLD/일반 토글)
맑은 고딕 (12pt)             -- 각주, 꺾쇠 제목, 박스 내 본문
돋움체 (10pt)                -- 담당부서 표
```

**암묵 규칙**: 본문 폰트(바탕)와 보조 폰트(맑은 고딕)의 크기 차이는 정확히 2pt (14pt vs 12pt). 이 2pt 차이가 "본문"과 "보조 정보"를 시각적으로 구분한다.

## III. XML 구현 관점의 핵심 패턴

### 1. Bold 강조의 XML 구조

한 문장 안에서 일반-굵게가 반복적으로 전환되는 것이 이 문서의 핵심 특성이다:

```xml
<hp:run charPrIDRef="40"><hp:t>  중소벤처기업부(장관 한성숙, 이하 중기부)는 </hp:t></hp:run>
<hp:run charPrIDRef="47"><hp:t>예비창업자</hp:t></hp:run>
<hp:run charPrIDRef="40"><hp:t>와</hp:t></hp:run>
<hp:run charPrIDRef="47"><hp:t> 창업기업</hp:t></hp:run>
<hp:run charPrIDRef="40"><hp:t>의 </hp:t></hp:run>
<hp:run charPrIDRef="47"><hp:t>고민</hp:t></hp:run>
<hp:run charPrIDRef="40"><hp:t>을 </hp:t></hp:run>
<hp:run charPrIDRef="47"><hp:t>한 곳</hp:t></hp:run>
<hp:run charPrIDRef="40"><hp:t>에서 </hp:t></hp:run>
<hp:run charPrIDRef="47"><hp:t>상담</hp:t></hp:run>
<hp:run charPrIDRef="40"><hp:t>받을 수 있는 </hp:t></hp:run>
<hp:run charPrIDRef="47"><hp:t>'온라인 스타트업 원스톱 지원센터'를 3월 30일(월) 개소</hp:t></hp:run>
<hp:run charPrIDRef="40"><hp:t>한다고 밝혔다.</hp:t></hp:run>
```

- charPrIDRef="40" = 14pt 바탕 일반 (본문 기본)
- charPrIDRef="47" = 14pt 바탕 BOLD (본문 강조)
- charPrIDRef="48" = 14pt 바탕 BOLD (동일 역할, 다른 ID -- 미세 속성 차이)

**핵심 원리**: charPr은 두 개만 교대하면 된다. 동일 문단 내에서 일반/BOLD를 반복 전환.

### 2. 소제목 인용 (밑줄+굵게)의 XML 구조

```xml
<hp:run charPrIDRef="61"><hp:t> "원스톱 상담 수요 확인</hp:t></hp:run>
<hp:run charPrIDRef="66"><hp:t> … </hp:t></hp:run>
<hp:run charPrIDRef="62"><hp:t>4개월간 상담 7천여 건</hp:t></hp:run>
<hp:run charPrIDRef="61"><hp:t>"</hp:t></hp:run>
```

- charPr[61]: 한컴 윤고딕 230, 14pt, BOLD, underline=BOTTOM shape=SOLID color=#000000
- charPr[66]: 한컴 윤고딕 230, 14pt, BOLD (밑줄 없음)
- charPr[62]: = charPr[61]과 동일 스펙

### 3. 개요 박스 (둥근 점선 테두리)의 XML 구조

```xml
<hp:tbl id="2112896569" ... borderFillIDRef="12" noAdjust="0">
  <!-- borderFill[12]: CIRCLE 0.3mm 검정 (외곽 둥근점선) -->
  <hp:sz width="47634" ... />
  <hp:outMargin left="141" right="141" top="141" bottom="141"/>
  <hp:inMargin left="510" right="510" top="141" bottom="141"/>
  <hp:tr>
    <hp:tc ... borderFillIDRef="13">
      <!-- borderFill[13]: DASH 0.12mm 검정 (내부 점선) -->
      <hp:subList ... vertAlign="CENTER">
        <hp:p ...>
          <hp:run charPrIDRef="7"><hp:t> ‣ </hp:t></hp:run>
          <hp:run charPrIDRef="15"><hp:t>(추진 배경)</hp:t></hp:run>
          ...
        </hp:p>
      </hp:subList>
    </hp:tc>
  </hp:tr>
</hp:tbl>
```

### 4. 상담 사례 박스 (장식선 테두리)의 XML 구조

장식선 효과는 3행 3열 표의 셀 테두리 조합으로 구현:

```
행0: [빈셀(하단선만)] [제목 셀(테두리없음, rowSpan=2)] [빈셀(하단선만)]
행1: [빈셀(좌상만)]   [     -- 병합 --             ] [빈셀(우상만)]
행2: [     전체 병합 (colSpan=3), 좌우하 0.2mm       ]
```

- borderFill[33]: 하단만 SOLID 0.2mm = 제목 양쪽의 가로선
- borderFill[34]: 테두리 전체 NONE = 제목 셀 (테두리 없이 텍스트만)
- borderFill[37]: 좌우 SOLID 0.2mm + 하단 SOLID 0.2mm = 본문 영역

이것은 "가로선-제목-가로선" 장식을 순수 표 셀 테두리만으로 구현한 기법이다.

### 5. 참고 헤더 (남색 배경)의 XML 구조

```xml
<hp:tbl ... borderFillIDRef="27">
  <hp:tr>
    <hp:tc ... borderFillIDRef="28">
      <!-- borderFill[28]: SOLID 0.3mm + 배경 #000080 (남색) -->
      <hp:run charPrIDRef="49"><hp:t>참고 1</hp:t></hp:run>
      <!-- charPr[49]: 17pt HY헤드라인M (흰색 텍스트는 별도 속성) -->
    </hp:tc>
    <hp:tc ... borderFillIDRef="29">
      <!-- 구분선 역할 빈 셀 (폭 565, 좌우 0.3mm만) -->
    </hp:tc>
    <hp:tc ... borderFillIDRef="30">
      <!-- borderFill[30]: SOLID 0.3mm + 배경 #FFFFFF -->
      <hp:run charPrIDRef="50"><hp:t> ｢스타트업 원스톱 지원센터｣ 인포그래픽</hp:t></hp:run>
    </hp:tc>
  </hp:tr>
</hp:tbl>
```

## IV. 종합: 이 문서의 디자인 철학

### 1. "스캔 가능성" 최우선
- Bold는 문장의 30~40%를 차지하며, 기자가 기사를 쓸 때 Bold만 훑어도 핵심이 파악되도록 설계
- 소제목 인용이 문서를 3개 섹션으로 명확히 구분

### 2. "테두리 = 의미" 원칙
- 점선 둥근테두리 = 보조 정보, 실선 = 정형 데이터, 장식선 = 스토리
- 배경색은 표 헤더와 참고 라벨에만 극도로 제한적으로 사용

### 3. "폰트 크기 = 정보 계층" 원칙
- 22pt (제목) > 17pt (참고 제목) > 16pt (참고 부제) > 14pt (본문) > 12pt (각주/박스) > 10pt (담당부서)
- 크기 차이가 항상 2pt 이상이어서 시각적 위계가 명확

### 4. "일관된 여백과 리듬"
- 단락 시작은 항상 2칸 들여쓰기
- 소제목 전후 빈 줄 1개
- 각주는 본문과 같은 들여쓰기 + 작은 폰트
- 160% 줄간격으로 통일 (읽기 편한 간격)

### 5. "마커 없는 본문" 철학
- 본문 단락에서는 마커를 일절 사용하지 않음
- 구조는 오직 Bold/일반의 리듬과 들여쓰기로 만듦
- 마커는 박스, 개요, 열거 등 특수 영역에서만 사용
