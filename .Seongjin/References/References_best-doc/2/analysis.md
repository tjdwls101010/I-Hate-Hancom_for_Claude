## 2. 암묵지 (경험자만 아는 디자인 감각)

### 2.1 굵은 강조의 정교한 패턴

이 문서의 bold 사용은 무작위가 아니라 **정보 계층을 반영하는 정밀한 시스템**이다.

**규칙 1: 핵심 정책명/기관명만 굵게**
- "제18차 정보통신전략위원회" -- bold
- "과학기술정보통신부" -- bold
- "관계부처와 합동" -- bold
- 일반 설명문은 절대 bold하지 않음

**규칙 2: 핵심 정책 방향/결론부만 굵게**
- "디지털 경쟁력의 확보도 지속 요구" -- bold (단락의 결론)
- "국가 디지털 기반 시설(인프라) 확충" -- bold (전략 명칭)
- "디지털 역량 강화" -- bold (전략 명칭)
- "국가 지능 정보화 고도화" -- bold (전략 명칭)
- "디지털 포용환경 조성" -- bold (전략 명칭)
- "핵심과제로 추진" -- bold (행동 결론)

**규칙 3: 마감 단락의 전략적 bold**
- "매년 세부 시행계획을 수립" -- bold
- "범부처 협력 체계" -- bold
- "'28년까지 대한민국 디지털 기반 시설(인프라) 및 경쟁력 확보를 위한 이정표" -- bold (클라이맥스)
- "신규 위촉된 제6기 민간위원" -- bold
- "13명의 민간 전문가" -- bold

**왜 이렇게 하는가**: 보도자료는 기자가 기사를 작성할 때 참고하는 문서다. bold는 "기자가 기사 제목/리드에 뽑아야 할 핵심 키워드"를 표시하는 역할을 한다. 공무원은 기자의 시선으로 역설계하여 bold를 배치한다. 또한 상급자가 2분 안에 훑어볼 때 bold만 읽어도 정책의 뼈대가 파악되도록 설계되어 있다.

### 2.2 ▲ 마커와 동그라미 번호(①②③④)의 이중 구조

이 문서의 가장 독특한 구조화 방식:

**XML 구현**:
```xml
<hp:run charPrIDRef="50"><hp:t>▲</hp:t></hp:run>
<hp:run charPrIDRef="98"><hp:t>①</hp:t></hp:run>
<hp:run charPrIDRef="111"><hp:t>연결망(</hp:t></hp:run>
```
- ▲ 마커: 14pt 바탕(charPr id=50, 86) -- 일반 굵기, 자간 -1~-2
- ① 번호: 14pt 바탕(charPr id=98, 99, 104, 105) -- Bold 또는 일반
- 본문 텍스트와 별도의 charPr로 분리

**이중 번호 체계**:
본문에서 ▲①, ▲②, ▲③, ▲④로 4개 전략을 나열한 후, 그 아래에서 각각 (①...), (②...), (③...), (④...)로 상세 설명한다.

- 요약(overview)에서: `▲①연결망... 확충 ▲②디지털... 강화 ▲③산업... 고도화 ▲④디지털... 조성`
- 상세 설명에서: `(①디지털 기반 시설<인프라> 확충)`, `(②디지털 역량 강화)` 등

**왜 이렇게 하는가**: ▲는 시각적 "총알" 역할로, 빠르게 스캔하는 독자의 눈을 멈추게 한다. ①은 순서/체계를 부여한다. 이 이중 구조는 "한 번 요약 + 한 번 상세"라는 보도자료 특유의 서술 패턴을 시각적으로 지원한다.

### 2.3 (①제목) 형식의 소제목 패턴

```
(①디지털 기반 시설<인프라> 확충) [이후 설명]
(②디지털 역량 강화) [이후 설명]
(③국가 지능 정보화 고도화) [이후 설명]
(④디지털 포용환경 조성) [이후 설명]
```

**XML 구현**:
```xml
<hp:run charPrIDRef="103"><hp:t>(</hp:t></hp:run>     <!-- 14pt 바탕 Bold -->
<hp:run charPrIDRef="109"><hp:t>①</hp:t></hp:run>     <!-- 14pt 바탕 Bold -->
<hp:run charPrIDRef="110"><hp:t>디지털 기반 시설&lt;인프라&gt; 확충)</hp:t></hp:run> <!-- 14pt 바탕체 Bold -->
<hp:run charPrIDRef="102"><hp:t> </hp:t></hp:run>     <!-- 14pt 바탕체 일반 (공백) -->
<hp:run charPrIDRef="92"><hp:t>전국 5세대...</hp:t></hp:run>  <!-- 14pt 바탕체 일반 (본문) -->
```

핵심: 괄호 + 번호 + 소제목은 **모두 Bold**이고, 괄호 닫힌 후 공백 하나를 두고 본문이 일반체로 시작된다. 소제목과 본문이 같은 단락에 이어지므로, 별도의 줄바꿈 없이 밀도를 유지한다.

**왜 이렇게 하는가**: 정부 보도자료는 페이지 수를 절약해야 한다(관례적으로 본문 1~2페이지). 소제목을 별도 줄로 빼면 공간 낭비이므로, 인라인으로 처리하되 bold로 시각적 구분을 만든다.

### 2.4 단락 간 전환 장치

이 문서는 명시적인 소제목이나 구분선 없이 단락만으로 주제를 전환한다.

**전환 패턴**:
1. **빈 줄(8pt 바탕체) 삽입**: 주제가 바뀔 때마다 charPr id=87 (8pt 바탕체)의 빈 단락이 들어감
   - 이것은 `<hp:run charPrIDRef="87"/>` (텍스트 없는 run)
   - 8pt 높이의 빈 줄 = 약 2.8mm의 시각적 간격
2. **도입 접속사**: "그간", "이에 대응하여", "한편" 등으로 주제 전환을 신호
3. **기관명 bold**: 새 주제 시작 시 주어(기관명)를 bold 처리

**왜 이렇게 하는가**: 보도자료는 "기사체"를 따르므로 소제목을 쓰지 않는다. 기자가 읽는 것처럼 자연스럽게 흘러가되, 8pt 빈 줄이 "여기서 화제가 바뀐다"는 미세한 시각 신호를 준다. 이 8pt는 본문 줄간격(14pt x 137% = 약 19pt)의 절반도 안 되는 미세한 간격이지만, 숙련된 독자는 이를 무의식적으로 인지한다.

### 2.5 밀도와 가독성의 양립

이 문서는 본문 2페이지에 방대한 정보를 담고 있다. 가독성을 유지하는 장치들:

1. **줄간격의 이중 구조**: 1페이지(도입+요약) = 137%, 2페이지(상세) = 152%
   - 1페이지는 밀도 높게(정보 전달 우선)
   - 2페이지는 약간 넉넉하게(읽기 편의)

2. **자간(spacing)의 미세 조정**:
   - bold 텍스트에 자간 축소(spacing=-1~-18) 적용
   - "국가 디지털 기반 시설(인프라)" 같은 긴 bold 구절이 줄을 넘기지 않도록 자간을 미세하게 좁힘
   - charPr id=58: spacing=-14, id=67: spacing=-18 등

3. **8pt 빈 줄 간격 조절**: 단락 사이 간격이 margin이 아니라 빈 단락으로 처리됨
   - 이는 HWPX 스타일 시스템의 한계를 우회하는 관행적 방법
   - 빈 단락의 높이를 8pt로 설정하여 간격 크기를 제어

4. **들여쓰기는 공백 문자**: indent가 0이고, 대신 전각 공백 2~3개로 들여쓰기 표현
   - 이는 한글 워드프로세서의 오래된 관행

### 2.6 「 」 와 법률명 표기

```
「제4차 정보 통신 진흥 및 융합 활성화 기본계획 및 제8차 지능정보사회 종합계획('26~'28)」
```

- 낫표 「 」: 법률명이나 공식 계획명을 인용할 때 사용
- 연도 표기: `'26~'28` (작은따옴표+2자리 연도, 물결표 연결)
- 기관 축약: "이하 '과기정통부'" -- 작은따옴표 사용
- 괄호 보충: "확정(서면 의결)", "연결망(네트워크)", "기반(플랫폼)"

**각주 표기법**:
- `정보통신전략위원회*` -- 본문 내 별표(*)
- `* 정보통신전략위원회 : ...` -- 별표 + 콜론으로 각주 시작
- 각주 텍스트는 12pt (본문 14pt보다 작음)
- 내어쓰기 적용 (intent=-2410)

### 2.7 숫자와 단위의 표기법

- "30여 년간" -- 숫자+한글 조사
- "총 25인으로" -- 숫자+한자어 단위
- "13명의" -- 숫자+한글 단위
- "2030년" -- 4자리 연도
- "'26~'28" -- 축약 연도 (각주/괄호 안에서)
- "5세대", "6세대" -- 숫자+세대
- 전화번호: "(044-202-6120)" -- 괄호 안 하이픈 구분

### 2.8 중점(‧)의 체계적 사용

```
심의‧의결
인공지능‧소프트웨어
석‧박사
공공‧민간
재난‧산업‧생활
통신‧디지털
```

- 가운뎃점(‧, U+2027)이 아니라 중점(‧, U+2027 또는 유사 문자)
- XML에서 `<hp:fwSpace/>‧<hp:fwSpace/>` 패턴: 중점 앞뒤에 전각 공백(fwSpace) 삽입
- 그러나 일부에서는 fwSpace 없이 직접 연결: "심의‧의결"
- **규칙**: 짧은 병렬어(2~3음절)에 중점 사용, 긴 구절 병렬에는 쉼표(,) 사용

### 2.9 본문과 별첨(붙임)의 시각적 구분

**물리적 구분**:
- 본문 2페이지 끝 → 페이지 넘김(pageBreak=1) → 붙임1 시작
- 본문은 텍스트 중심, 붙임은 인포그래픽 이미지

**붙임 헤더 테이블 디자인**:
- 3열: "붙임1" (반전 배경, borderFillIDRef=17) | 구분선 (borderFillIDRef=18, 폭 565로 매우 좁음) | 제목 (borderFillIDRef=19)
- "붙임1"은 별도의 강한 배경색/테두리로 라벨 역할
- 좁은 2열이 시각적 구분선 역할

**왜 이렇게 하는가**: 정부 보도자료에서 "붙임"은 본문보다 상세한 데이터를 제공하는 부록이다. 기자는 보통 본문만 읽고 기사를 쓰지만, 깊이 있는 기사를 쓸 때 붙임을 참조한다. 따라서 본문과 붙임은 시각적으로 명확히 분리되어야 하며, 이 문서는 페이지 넘김 + 독특한 붙임 헤더 디자인으로 이를 달성한다.

### 2.10 담당부서 표의 위치와 의미

- 본문 마지막, 붙임 직전에 배치
- 이것은 "이 보도자료에 대한 문의처"
- 6열 2행: 부서명(실+과) | 책임자(과장)+전화 | 담당자(사무관)+전화
- **배포 시 핵심 정보**: 기자가 추가 취재할 때 연락할 곳

### 2.11 문서 하단 로고 영역

2페이지 하단에 사각형(rect) 도형 안에 3개 이미지 배치:
1. "내일을 만드는 과학기술, 내 삶을 채우는 디지털AI" 슬로건 이미지
2. 정책브리핑(korea.kr) 로고 (하이퍼링크 연결)
3. 공공누리/공공저작물 자유이용허락 마크

이것은 과기정통부 보도자료 템플릿의 표준 푸터로, 모든 보도자료에 동일하게 들어간다.

## 3. XML 구현 관점의 핵심 발견

### 3.1 메타태그(metatag) 시스템

이 문서는 템플릿 기반으로, 주요 셀에 JSON 메타태그가 있다:
```
metatag="{\"name\":\"#기관명\"}"
metatag="{\"name\":\"#보도일시\"}"
metatag="{\"name\":\"#배포일시\"}"
metatag="{\"name\":\"#제목명\"}"
metatag="{\"name\":\"#부제목명\"}"
metatag="{\"name\":\"#본문\"}"
metatag="{\"name\":\"#담당부서\"}"
metatag="{\"name\":\"#책임자명\"}"
metatag="{\"name\":\"#담당자명\"}"
```
이는 정부 보도자료 양식(서식)의 입력 필드를 정의하는 시스템이다.

### 3.2 ClickHere 필드

본문은 `fieldBegin type="CLICK_HERE"`로 시작:
```xml
<hp:fieldBegin id="1999385510" type="CLICK_HERE" name="본문" editable="1">
  <hp:stringParam name="Direction">이곳을 마우스로 누르고 본문 내용을 입력하세요.</hp:stringParam>
</hp:fieldBegin>
```
본문 전체가 하나의 "클릭하여 입력" 필드 안에 들어있다.

### 3.3 Bold 구현: 자식 요소 방식

HWPX에서 bold는 charPr의 속성이 아니라 **빈 자식 요소**로 구현:
```xml
<hh:charPr id="58" height="1400" ...>
  <hh:fontRef hangul="5" .../>
  ...
  <hh:bold/>          <!-- 이 빈 요소의 존재 = Bold -->
  <hh:underline type="NONE" .../>
</hh:charPr>
```

### 3.4 fwSpace (전각 공백)

중점(‧) 앞뒤의 공백은 `<hp:fwSpace/>`로 구현:
```xml
<hp:t>인공지능<hp:fwSpace/>‧<hp:fwSpace/>소프트웨어</hp:t>
```
이는 일반 공백보다 넓은 전각 공백으로, 중점을 시각적으로 분리한다.

### 3.5 charPr의 과다한 분화

본문 텍스트에 사용된 charPr ID가 47, 50, 51, 52, 53, 55, 56, 58, 63, 70, 86, 89, 91, 92, 94, 95, 97, 100, 101, 103, 104, 105, 106, 107, 109, 110, 111, 119, 120, 121 등 **30개 이상**에 달한다. 이는 한글 워드프로세서의 특성으로, 같은 "14pt 바탕체"라 해도 자간(spacing)이 -1, -2, -14, -18 등으로 미세하게 다른 여러 charPr이 존재한다. 이는 편집자가 줄 맞춤을 위해 수동으로 자간을 조정한 흔적이다.

## 4. 종합: 이 문서의 디자인 원칙

1. **2-페이지 규율**: 본문은 반드시 2페이지 이내. 나머지는 "붙임"으로.
2. **Bold = 기사 키워드 가이드**: 기자가 뽑아야 할 핵심어를 bold로 안내.
3. **▲① 이중 구조**: 먼저 전체를 요약(▲①~④)하고, 다시 각각을 상세 설명((①)~(④)).
4. **인라인 소제목**: 별도 줄을 쓰지 않고, (①제목) 형식으로 본문과 같은 줄에 배치.
5. **8pt 빈 줄 = 화제 전환**: 명시적 소제목 대신 미세한 공간으로 주제 전환을 신호.
6. **자간의 미세 조정**: 줄 끝 정렬과 시각적 밀도를 위해 charPr별 자간을 수동 조절.
7. **템플릿 + 메타태그**: 표준 서식의 metatag 시스템 위에 내용을 채우는 구조.
8. **밀도 vs 가독성의 절충**: 줄간격(137~152%), 자간(-1~-18), 8pt 빈 줄을 조합하여 빽빽하지만 읽히는 문서를 달성.
