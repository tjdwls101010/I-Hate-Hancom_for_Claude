## 2. 암묵지 (경험적 디자인 감각)

### 2.1 들여쓰기와 * 주석의 관계

본문과 * 주석은 **동일한 좌측 기준선에서 시작**한다. 둘 다 "  "(2칸 스페이스)로 시작하며, paraPrIDRef="0" (들여쓰기 0)을 공유한다. 차이는 오직 **글자 크기**(14pt vs 12pt)뿐이다.

구체적 패턴:
```
[본문 14pt]   기후에너지환경부(장관 김성환)는 ... 신청(1차*)을 받는다.
[빈줄 3pt]
[주석 12pt]   * 2026년 상·하반기로 나눠 2회 신청·접수, 2차 모집기간 7.20~8.5 예정
```

이것이 만드는 **시각적 효과**: 주석은 본문보다 2pt 작기 때문에 "부수적 설명"으로 자연스럽게 인식된다. 별도의 들여쓰기를 하지 않아도 글자 크기 차이만으로 위계가 성립한다.

반면 붙임 1의 * 주석은 다르다:
```
[본문 15pt]  ㅇ (규모) 11,000명* 대상, 1인당 지원액은 10만원 산정
[빈줄 3pt]
[주석 13pt]      * 상반기(1차) 8,000명 / 하반기(2차) 3,000명
```
여기서 주석은 "    * "(4칸 스페이스)로 시작하여, ㅇ 불릿 항목보다 안쪽으로 **명시적 들여쓰기**가 있다. 이는 불릿 구조에서 주석이 해당 불릿의 하위 설명임을 보여주기 위한 것이다.

**원칙**: 본문 보도자료에서는 동일 레벨 들여쓰기 + 글자 크기 차이로 구분. 구조화된 붙임에서는 상위 불릿보다 안쪽 들여쓰기를 추가.

### 2.2 다중 별표(*, **, ***) 주석의 디자인

이 문서는 *, **, *** 삼중 주석 시스템을 사용한다:

```
상품*, 서비스** 및 건강나누리캠프 교육*** 등을 이용할 수 있으며...
```

대응하는 주석:
```
* 아토피로션, 곰팡이제거제 및 진드기 제거 세제 등 / **곰팡이 제거, 침구 살균·세탁 등
/ ***국립공원 등에서 환경성질환 예방·관리 교육 및 자연환경 체험 서비스 등
```

특징:
- 본문의 *는 `<hh:supscript/>`(위첨자)로 처리 (charPrIDRef="107", "127" 등)
- 주석 텍스트에서 *, **, ***는 위첨자가 아닌 일반 텍스트
- 여러 주석을 " / "로 연결하여 한 문단에 배치
- 줄바꿈이 필요한 경우 `<hp:lineBreak/>` 사용

### 2.3 굵은 강조(bold) 패턴 분석

**본문 (보도자료 1~2페이지)**:
- 본문에는 **bold가 전혀 사용되지 않는다**. 모든 텍스트가 regular weight의 바탕체 14pt이다.
- 자간(spacing) 미세조정(-9~+6)으로 양쪽정렬을 맞추지만, 이는 강조가 아닌 조판 기술이다.

**붙임 1 (사업 개요)**:
- 섹션 제목(□ 배경)의 HY헤드라인M은 bold가 아닌 서체 자체의 두께
- 불릿 항목에서 **선택적 bold** 사용:
  - "(대상)", "(규모)", "(지원내용)" 등 **괄호 키워드**는 bold (charPrIDRef="47": 휴먼명조 15pt bold, spacing=-3)
  - "기초생활수급자 또는 차상위계층 어린이"처럼 **핵심 수혜 대상**은 bold
  - "11,000명* 대상, 1인당 지원액은 10만원"에서 **숫자와 금액**은 bold
  - 일반 설명 텍스트는 regular

**붙임 3 (Q&A)**:
- 질문: HY헤드라인M (자체가 굵은 서체)
- 답변: 맑은 고딕 15pt, bold 대신 **밑줄(underline)**로 강조
  - "취약계층 어린이에게 환경성질환" -- 밑줄
  - "기초생활수급자 또는 차상위계층 어린이" -- 밑줄
  - "1차 신청기간은 3월 30일부터 4월 15일까지" -- 밑줄
  - "어린이와 함께 거주" -- 밑줄

**원칙 요약**:
| 영역 | 강조 방식 |
|------|----------|
| 보도자료 본문 | 강조 없음 (서체 크기와 문맥으로 전달) |

- col1: 1행 좌우+상+없음(하), 2행 좌우+없음(상)+하, 3행 좌우+상+없음(하), 4행 좌우+없음(상)+하 → 기관 쌍 단위로 상하 구분

### 2.7 Q&A 표의 초록색 테두리 구현 (XML 관점)

Q&A의 각 답변 상자는 2행 2열 표로 구현되며, 초록색(#008000) 0.5mm 테두리를 사용한다.

```xml
<!-- borderFillIDRef="19" - A 셀 상단: 좌+상 초록, 우+하 없음 -->
<hh:borderFill id="19">
  <hh:leftBorder type="SOLID" width="0.5 mm" color="#008000"/>
  <hh:rightBorder type="NONE" width="0.5 mm" color="#008000"/>
  <hh:topBorder type="SOLID" width="0.5 mm" color="#008000"/>
  <hh:bottomBorder type="NONE" width="0.5 mm" color="#008000"/>
</hh:borderFill>

<!-- borderFillIDRef="20" - 답변 셀: 우+상+하 초록, 좌 없음 -->
<hh:borderFill id="20">
  <hh:leftBorder type="NONE" width="0.5 mm" color="#008000"/>
  <hh:rightBorder type="SOLID" width="0.5 mm" color="#008000"/>
  <hh:topBorder type="SOLID" width="0.5 mm" color="#008000"/>
  <hh:bottomBorder type="SOLID" width="0.5 mm" color="#008000"/>
</hh:borderFill>

<!-- borderFillIDRef="21" - A 셀 하단: 좌+하 초록, 우+상 없음 -->
<hh:borderFill id="21">
  <hh:leftBorder type="SOLID" width="0.5 mm" color="#008000"/>
  <hh:rightBorder type="NONE" width="0.5 mm" color="#008000"/>
  <hh:topBorder type="NONE" width="0.5 mm" color="#008000"/>
  <hh:bottomBorder type="SOLID" width="0.5 mm" color="#008000"/>
</hh:borderFill>
```

이 3개의 borderFill을 조합하면:
- "A" 글자가 좌상단 작은 셀에 위치
- 답변 텍스트가 우측 큰 셀에 위치 (col1은 rowSpan="2"로 병합)
- 결과적으로 초록색 "L자" 모양 테두리가 A를 감싸고, 답변은 완전한 사각형으로 둘러싸임

### 2.8 서체 사용 체계

이 문서는 **4개 서체 축**을 사용한다:

| 서체 | 역할 | 사용 위치 |
|------|------|----------|
| **바탕** (fontRef=2) | 보도자료 본문 | 1~2페이지 전체 본문, 주석 |
| **함초롬돋움** (fontRef=4) | 제목/강조 | 제목 표의 주제목 26pt bold |
| **HY헤드라인M** (fontRef=7) | 구조 라벨 | 붙임 헤더, Q&A 질문, □ 섹션 |
| **휴먼명조** (fontRef=8) | 붙임 본문 | 사업 개요의 ㅇ 불릿 본문 |
| **맑은 고딕** (fontRef=1) | Q&A 답변, 주석 | 붙임3 답변 텍스트, 일부 주석 |
| **돋움체** (fontRef=0) | 메타 정보 | 담당부서 표, 보도시점 정보 |
| **함초롬바탕** (fontRef=5) | 부제목 | 제목 표의 부제목, 기타 |

### 2.9 빈 줄의 전략적 사용

이 문서는 빈 줄(empty paragraph)을 **3가지 크기**로 사용한다:

1. **3pt 빈 줄** (height=300): 본문 문단과 * 주석 사이, 또는 문단 내 미세 간격
   - charPrIDRef="115" (바탕 3pt) 또는 charPrIDRef="118" (바탕 3pt, spacing=2)

2. **5pt 빈 줄** (height=500): 붙임의 불릿 항목과 주석 사이
   - charPrIDRef="54" (휴먼명조 5pt)

3. **기본 빈 줄** (해당 서체 기본 크기): 문단 간 일반 간격
   - 본문에서 14pt 바탕의 빈 줄 = 약 14pt 높이

**원칙**: 빈 줄의 높이를 의도적으로 조절하여, 문단 사이 공간을 정밀하게 제어한다. paraPr의 prev/next margin과 결합하여 사용.

### 2.10 양쪽정렬에서의 자간 미세조정 기법

본문의 바탕 14pt 텍스트에서 관찰되는 charPr의 spacing 값:

| charPrIDRef | spacing | 용도 |
|-------------|---------|------|
| 13 | -1 | 약간 빽빽한 줄 |
| 14 | +2 | 약간 넓은 줄 |
| 15 | 0 | 표준 |
| 16 | +3 | 넓은 줄 |
| 17 | -2 | 빽빽한 줄 |
| 18 | -3 | 매우 빽빽한 줄 |
| 19 | -4 | 가장 빽빽한 줄 |

이 자간 변동은 **한 문단 내에서도 run 단위로 다르게 적용**된다. 예를 들어:

```
<hp:run charPrIDRef="121"><hp:t>  2026년도 환경보건이용권 지원은...</hp:t></hp:run>  (spacing=-8)
<hp:run charPrIDRef="125"><hp:t> </hp:t></hp:run>  (spacing=-5)
<hp:run charPrIDRef="18"><hp:t>/ </hp:t></hp:run>  (spacing=-3)
```

이것은 한컴의 **양쪽정렬 자동 자간 조정** 기능이 저장 시 고정값으로 기록된 것이다. 수동으로 재현할 필요는 없으며, 양쪽정렬 설정 시 자동으로 처리된다.

### 2.11 보도자료 본문의 서술 흐름 패턴

이 문서의 본문 구조:

1. **도입 문단**: "[부처명](장관 [이름])는 [날짜]까지 ... '2026년 [정책명]' 신청(1차*)을 받는다."
   - 첫 문단에서 누가, 언제, 무엇을 하는지 명시

2. **배경 설명**: "지난해(2025년) 처음으로 도입된 ... 사업이다."
   - 정책의 역사와 맥락

3. **신청 방법**: "환경보건이용권을 지원 받으려면 ... 신청*하면 된다."
   - 구체적 행동 안내

4. **세부 내용**: "2026년도 환경보건이용권 지원은 ... 구분된다."
   - 상세 구조 설명

5. **인용 마무리**: "[직함]은 "..." 라며, "..."라고 밝혔다."
   - 정부 관계자 발언으로 마무리

6. **붙임 목록**: "붙임  1. ... 2. ... 3. ..."

7. **담당부서 정보표**

### 2.12 날짜 표기 규칙

- 보도시점: "2026. 3. 29.(일) 12:00 (월요일 조간)"
  - 연. 월. 일.(요일) 시:분 (요일 조/석간)
  - 숫자 사이 스페이스+마침표+스페이스
- 배포: "2026. 3. 27.(금)"
  - 연. 월. 일.(요일)
- 본문 내: "3월 30일부터 4월 15일까지"
  - 한글 "월", "일" 사용
- 주석 내: "7.20~8.5"
  - 축약 표기 (마침표 구분, ~로 기간)
