# 보도자료 분석: 서울과 여의도 윤중로 벚꽃 개화

**문서**: 수도권기상청 보도자료 (2026. 3. 29.)
**특징**: 사진(이미지) 삽입형 보도자료 -- 일반 텍스트 전용 보도자료와 구별되는 레이아웃

## 1. 형식지 (명시적 규격)

### 1.1 문서 기본 구조

| 영역 | 내용 |
|------|------|
| **헤더** | 기관 로고(수도권기상청) + "보도자료" + 보도시점/배포 일시 |
| **제목** | 테두리 박스 안에 대제목 + 부제목 |
| **본문** | 마커(사각/원) 기반 계층 구조 |
| **사진 영역** | 2열 사진 + 캡션 (테이블 내 삽입) |
| **담당자 정보** | 하단 표 (담당 부서, 책임자, 담당자, 연락처) |
| **붙임** | 별도 페이지에 역사 데이터 표 |

### 1.2 제목 스타일

- **대제목**: "오늘(29일), 서울과 여의도 윤중로 벚꽃 개화"
  - 테두리 박스(borderFill) 내부에 중앙 정렬
  - 26pt 함초롬돋움 Bold, 자간 -12%
  - 밑줄 없음, 검정색
- **부제목**: "- 작년보다 6일 빠르고, 평년보다 10일 빠름"
  - 같은 박스 안, 중앙 정렬
  - 16pt HY헤드라인M, 자간 -5%
  - 하이픈(-)으로 시작하는 부연 설명

### 1.3 본문 스타일

- **기본 글꼴**: 바탕체 14pt, 줄간격 160~180%, 양쪽 정렬
- **괄호 안 부연 정보**: 12pt로 축소 (예: "(국립기상박물관, 서울 종로구 송월길 52)")
- **URL**: 12pt 바탕체 (예: "(www.weather.go.kr)")
- **주석(※)**: 12pt 돋움체 -- 본문 바탕체와 구별하여 주석임을 시각적으로 표시

### 1.4 이미지 삽입 구조 (XML 관점)

사진은 **테이블(hp:tbl) 안에 삽입**되는 구조:

```
<hp:tbl> (테두리 없음, 2열 구조)
  <hp:tr>
    <hp:tc> [왼쪽 사진 - hp:pic, treatAsChar="1", textWrap="TOP_AND_BOTTOM"] </hp:tc>
    <hp:tc> [오른쪽 사진 - hp:pic, 동일 구조] </hp:tc>
  </hp:tr>
  <hp:tr>
    <hp:tc colspan="2"> [캡션 텍스트 - 중앙 정렬] </hp:tc>
  </hp:tr>
</hp:tbl>
```

- **hp:pic 속성**: `numberingType="PICTURE"`, `textWrap="TOP_AND_BOTTOM"`, `textFlow="BOTH_SIDES"`
- **hc:img**: `binaryItemIDRef="image2"` 등으로 BinData 폴더의 이미지 참조
- **pos**: `treatAsChar="1"` (문자처럼 취급), `horzRelTo="COLUMN"` (단 기준 정렬)
- **사진 크기**: curSz width=22397, height=16806 (약 7.9cm x 5.9cm, 2장 나란히)
- **캡션**: 테이블 하단 행에 병합 셀, 중앙 정렬, 14pt 바탕체

## 2. 암묵지 (경험적 디자인 감각)

### 2.1 굵은 강조의 정밀한 문법

이 문서의 bold 사용은 무작위가 아니다. 정확한 패턴이 있다:

**패턴 A: 핵심 사실(날짜+사건) 묶음 강조**
> "올해 **서울의 벚꽃이 3월 29일(일) 개화**했다고 발표했다."

- `서울의 벚꽃이 3월 29일(일)` = charPr 98 (14pt 바탕체 **bold**, 자간 -6)
- `개화` = charPr 98 (동일 bold)
- `했다고` = charPr 97 (14pt 바탕체 normal)

원칙: **주어 + 날짜 + 핵심 동사**를 하나의 bold 블록으로. "했다고 발표했다"는 bold에서 빠진다. 이것은 "무엇이 언제 어떻게 되었는가"만 시각적으로 포착되게 하는 기법이다.

**패턴 B: 두 번째 주요 사실에서도 동일 구조**
> "서울의 **대표 벚꽃 군락 단지인 여의도 윤중로의 벚꽃도 3월 29일(일) 개화**했다."

여기서도 "대표 벚꽃 군락 단지인..."이라는 수식어까지 bold에 포함시켜, 이것이 단순 반복이 아닌 별도 관측 지점임을 강조한다.

**패턴 C: 배경 설명에서의 강조**
> "**서울의 벚꽃 개화**는 서울기상관측소..."
> "**벚꽃과 철쭉 군락지 개화 현황을 제공**하고 있다."

배경 설명에서도 핵심 키워드(무엇에 관한 이야기인가)를 bold로, 나머지는 normal.

**패턴 D: 비교 수치는 강조하지 않음**
> "이는 작년(4월 4일)보다 6일 빠르고, 평년(4월 8일)보다 10일 빠르다."

이 비교 문장은 전체가 normal. 숫자가 많아 bold까지 걸면 시각적 과부하. 대신 자간을 -11 ~ -17로 조여서 한 줄에 깔끔하게 들어가도록 조절한다.

### 2.2 사각(box)과 원(circle) 마커의 계층 차이

```
□  대주제 문단   (paraPr 66~68: intent=-2100~-2800, left=0, 줄간격 160~180%)
  ○  세부 설명   (paraPr 59~64: intent=-2185~-3686, 줄간격 180%)
```

**□(사각 마커)**의 역할:
- "새로운 화제의 시작"을 알린다
- 이 문서에서 4개: (1) 개화 발표, (2) 관측 기준 설명, (3) 날씨누리 제공 안내, (4) 붙임 안내
- paraPr에서 indent(내어쓰기)가 약 -2800 HWPUNIT: 마커가 본문 왼쪽 마진보다 바깥으로 돌출
- 마커 글자 자체는 14pt 바탕체 normal (charPr 96)

**○(원 마커)**의 역할:
- □ 아래의 부연, 보충, 하위 설명
- 앞에 두 칸 들여쓰기(공백 2개)로 시작: "  ○"
- 이 문서에서 4개: (1) 여의도 윤중로 추가 관측, (2) 2000년부터 지정, (3) 개화 기준 정의, (4) 제공 위치
- paraPr에서 줄간격이 180%로 □의 160%보다 넓다 -- ○ 문단이 독립적으로 읽히게 하기 위함

**○ 마커의 색상 변주**:
- 첫 번째 ○ 문단("또한, 서울의 대표...")의 마커는 charPr 74 = **파란색(#0000FF)** 14pt 바탕체
- 이것은 "이 부분이 새로운 관측 지점(여의도)에 관한 내용"임을 시각적으로 구별하는 의도
- 나머지 ○는 검정 마커

### 2.3 기상 데이터의 표현 공식

이 문서는 기상 데이터를 일관된 공식으로 표현한다:

**개화 발표 공식**:
```
[기관]은 올해 [장소]의 벚꽃이 [월 일(요일)] 개화했다고 발표했다.
이는 작년([과거 날짜])보다 [N]일 빠르고, 평년([평년 날짜])보다 [N]일 빠르다.
```

핵심 원칙들:
- **절대 날짜 + 괄호 요일**: "3월 29일(일)" -- 괄호 안 요일은 속보성 강조
- **비교 표현의 3항 구조**: 작년 대비 + 평년 대비, 각각 괄호 안에 정확한 날짜 포함
- **"빠르다/느리다"의 일관성**: 양수면 "빠르다", 음수면 "늦다"
- **부제목에서의 압축**: "작년보다 6일 빠르고, 평년보다 10일 빠름" -- 구체적 날짜 생략한 압축형

### 2.4 사진과 텍스트의 관계 -- 시각적 리듬

이 문서의 레이아웃에는 명확한 리듬이 있다:

```
[텍스트] □ 핵심 사실 + 비교 수치
[사진]   전경 | 근접  (2장 나란히)
[캡션]   [그림N] 설명
[텍스트] ○ 추가 사실 + 비교 수치
[사진]   전경 | 근접  (2장 나란히)
[캡션]   [그림N] 설명
[텍스트] □ 배경 설명
   ...
```

**핵심 원칙**:
1. **사실 먼저, 증거(사진) 나중**: 텍스트로 "개화했다"를 먼저 말한 뒤, 사진으로 증명
2. **전경+근접 쌍**: 한 장소당 반드시 2장. 왼쪽=전경(현장 분위기), 오른쪽=근접(꽃 상태). "좌: 전경, 우: 근접" 캡션으로 명시
3. **사진은 절대 단독으로 존재하지 않음**: 반드시 직전 텍스트의 시각적 근거로 기능
4. **사진 뒤에는 새로운 화제 시작**: 사진이 이전 화제의 마침표 역할

**캡션 스타일**:
```
[그림N] [장소명] 벚꽃 개화 사진([날짜]/ 좌: [설명], 우: [설명])
```
- 대괄호 번호 + 장소 + 촬영 날짜 + 좌우 구분
- 중앙 정렬, 14pt 바탕체
- 테이블 내부 병합 셀

### 2.5 사진의 XML 구현 -- 테이블 기반 레이아웃

사진이 있는 보도자료의 핵심적인 XML 구조:

```xml
<!-- 이미지 2장을 나란히 배치하는 테이블 -->
<hp:tbl>  <!-- 테두리 없음(borderFill: NONE) -->
  <hp:tr>
    <hp:tc>  <!-- 왼쪽 셀: 전경 사진 -->
      <hp:p>
        <hp:run>
          <hp:pic textWrap="TOP_AND_BOTTOM" numberingType="PICTURE">
            <hp:pos treatAsChar="1" horzRelTo="COLUMN"/>
            <hp:sz width="22397" height="16806"/>  <!-- 약 7.9cm x 5.9cm -->
            <hc:img binaryItemIDRef="image2"/>
          </hp:pic>
        </hp:run>
      </hp:p>
    </hp:tc>
    <hp:tc>  <!-- 오른쪽 셀: 근접 사진 -->
      <!-- 동일 구조 -->
    </hp:tc>
  </hp:tr>
  <hp:tr>
    <hp:tc colspan="2">  <!-- 캡션: 병합 셀 -->
      <hp:p align="CENTER">
        [그림1] 서울기상관측소 벚꽃 개화 사진(3월 29일(일)/ 좌: 전경, 우: 근접)
      </hp:p>
    </hp:tc>
  </hp:tr>
</hp:tbl>
```

주목할 점:
- **테이블은 레이아웃 도구**: 데이터 표시가 아닌, 2열 사진 배치를 위한 **보이지 않는 테이블**
- **hp:pic의 shapeComment**: 원본 파일명, 촬영 날짜, 카메라 정보 등 EXIF 메타데이터가 포함됨 (한글 고유 기능)
- **사진 크기 비율 보존**: orgSz(원본)와 curSz(현재) 비율이 동일하게 scaMatrix로 관리

### 2.6 자간(spacing)의 미세 조정 -- 줄맞춤의 기술

이 문서에서 눈에 띄는 것은 charPr마다 자간이 다르다는 것:

| charPr | 자간 | 사용 위치 | 의도 |
|--------|------|----------|------|
| 96, 34 | 0 | □, ○ 마커 | 마커는 기본 자간 |
| 97, 98 | -6 | 핵심 사실 문장 | 한 줄에 맞추기 위한 압축 |
| 65 | -17 | "발표했다." 부분 | 극단적 압축으로 줄바꿈 방지 |
| 76, 77 | -9 | ○ 본문 | 중간 수준 압축 |
| 80 | -15 | 비교 수치 | 숫자+괄호가 많아 강한 압축 |

이것은 경험 있는 공무원이 인쇄물에서 "이 문장이 저 줄에서 한 글자만 넘어가면 안 된다"고 판단하고 자간을 수동으로 조절한 흔적이다. 디지털 문서이지만 **인쇄 기반 사고**가 반영되어 있다.

### 2.7 붙임(첨부) 자료의 디자인

**붙임 표지**:
```
[진한 파란 배경(#00468C) 셀]  [세로선]  [제목]
  붙임 1 (흰색 글자)                    서울 벚꽃 개화일 및 만발일
```
- 왼쪽에 짙은 파란색 배경 + 흰색 글자 = 시각적 탭 역할
- 오른쪽에 제목 = 내용 설명
- borderFill id=13: faceColor="#00468C" -- 기상청 CI 컬러

**데이터 표**:
- 헤더 행: 연한 하늘색 배경(#E0F4F6), 0.4mm 굵은 테두리
- 데이터 행: 흰색 배경, 0.2mm 일반 테두리
- 가장 최근 연도(2026)의 데이터는 빨간색(#FF0000)으로 강조
- 5열 x 30행 = 1901~2026 전체 데이터를 한 페이지에
- 12pt 맑은고딕, 장평 80%, 자간 -6 ~ -10 = 극한의 압축으로 한 페이지 내 수용

### 2.8 색상의 의미 체계

| 색상 | 용도 |
|------|------|
| #000000 (검정) | 기본 본문 |
| #0000FF (파란) | ○ 마커에서 새로운 관측 지점 강조 |
| #FF0000 (빨강) | 현재 연도 데이터, 붙임 번호 색상 |
| #00468C (진한 파란) | 붙임 표지 배경 -- 기상청 CI |
| #E0F4F6 (연한 하늘) | 데이터 표 헤더 배경 |
| #FFFFFF (흰) | 진한 배경 위 글자 |

## 3. 이 문서의 고유한 특징 (사진 삽입형 보도자료)

### 3.1 일반 보도자료와의 차이

| 항목 | 일반(텍스트 전용) | 이 문서(사진 삽입형) |
|------|-----------------|-------------------|
| 페이지 수 | 1~2페이지 | 2+붙임 페이지 |
| 시각 자료 | 없음 또는 표 | 현장 사진 6장 |
| □/○ 뒤 흐름 | 텍스트 → 텍스트 | 텍스트 → 사진 → 텍스트 |
| 레이아웃 도구 | 들여쓰기만 | 숨은 테이블(보이지 않는 테두리) |
| 증거 방식 | 숫자/데이터로 증명 | 사진+숫자 이중 증거 |

### 3.2 사진 보도자료의 암묵적 규칙

1. **사진 촬영 시점 = 보도 시점**: EXIF 데이터가 "2026년 03월 29일 오후 13:53~14:44", 보도시점 "15:40" -- 촬영 후 약 1시간 만에 보도자료 작성/배포
2. **2대 이상 기기로 촬영**: 서울기상관측소는 iPhone 14 Pro, 여의도는 Galaxy S26 -- 복수 관측자의 현장 확인
3. **사진 해상도**: 원본 4032x3024 (약 12MP)를 7.9cm x 5.9cm로 축소 -- 인쇄용이 아닌 화면 확인용
4. **지도(그림3)는 단독 배치**: 현장 사진과 달리 설명용 지도는 1장으로, 테이블 1열 구조

## 4. HWPX 생성 시 주의사항

이 문서를 재현하거나 유사한 사진 삽입형 보도자료를 생성할 때:

1. **사진 삽입은 반드시 테이블 기반**: 직접 본문에 이미지를 넣지 않고, 보이지 않는 테이블(borderFill: NONE)로 레이아웃
2. **hp:pic의 treatAsChar="1"**: 이미지가 문자처럼 취급되어 본문 흐름을 따름
3. **textWrap="TOP_AND_BOTTOM"**: 이미지 좌우에 텍스트가 흐르지 않고, 위아래로만 배치
4. **캡션은 테이블 내부 병합 셀**: 별도 문단이 아니라 이미지 테이블의 하단 행
5. **shapeComment에 EXIF 정보 포함**: 한글 특유의 기능으로, 이미지 원본 정보를 XML에 기록
6. **자간 조정은 문단 단위가 아닌 run 단위**: 같은 문단 내에서도 run마다 다른 자간 적용
7. **붙임 표지의 색상 배경 셀**: borderFill에 faceColor 지정으로 구현
