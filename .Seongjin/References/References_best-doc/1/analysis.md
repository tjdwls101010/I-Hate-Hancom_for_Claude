## Part 2: 암묵지 (경험자만 아는 디자인 감각)

### 2.1 굵은 강조의 패턴: "성과 동사구만 굵게"

이 문서에서 볼드는 매우 절제적으로 사용된다. 전체 본문(약 15개 단락) 중 볼드가 적용된 곳은 단 3-4곳뿐이다.

**패턴 분석**:
- "**고품질 세포주를 만들고 품질시험을 완료**했다" -- 성과의 핵심 내용만 볼드. "-했다"는 제외.
- "**연구 초기 단계에서 필요한 비용과 시간을 줄일 수 있는 기반을 마련**했다" -- 기대효과를 볼드. 역시 "-했다"는 제외.
- 인용문 후반부의 핵심 메시지 -- "**한계를 해소하고 인공혈액을 포함한 세포치료제 등의 첨단바이오의약품 개발연구에 실질적인 도움이 될수 있도록 힘쓰겠다**"

**왜 이렇게 하는가**: 보도자료는 기자가 핵심을 빠르게 뽑아 기사화해야 한다. 볼드는 "이것이 뉴스 가치다"라는 신호다. 문장 전체를 굵게 하면 신호가 희석되므로, 정확히 "무엇을 달성했는가"를 나타내는 동사구(목적어+동사)만 굵게 처리한다. 동사 어미("-했다")를 볼드에서 제외하는 것은 시각적으로 문장이 자연스럽게 마무리되게 하기 위한 것이다.

**XML 구현**: 하나의 `<hp:p>` 내에서 여러 `<hp:run>`으로 분리. 볼드 run(charPrIDRef=102, bold 속성 포함)과 일반 run(charPrIDRef=101)을 교대 배치.

### 2.2 들여쓰기의 리듬: "전각 공백 2개의 일관성"

본문 단락의 첫 줄은 paraPr의 intent 값이 0인데도 불구하고, 텍스트 시작에 전각 공백 2개(`  `)가 삽입되어 있다. 이는 한컴 문서의 전통적 관행이다.

**관찰**:
- 일반 본문: `  ` (전각 공백 2개) + 본문 시작
- 인용문: `   ` (전각 공백 3개) + "본 연구팀은", "임승관 질병관리청장은"
- 주석(*): intent=-1273으로 내어쓰기, * 기호가 본문보다 왼쪽으로 돌출
- 임상등급 역분화줄기세포 정의: `  ` + 본문과 동일 레벨 들여쓰기

**왜 이렇게 하는가**: 인용문이 일반 본문보다 공백 1개 더 들어가는 것은 "이 단락은 다른 사람의 말"이라는 시각적 구분이다. 독자가 빠르게 스캔할 때 인용문을 식별할 수 있다. 주석의 내어쓰기는 * 기호를 눈에 띄게 만들어 "이것은 부연 설명"임을 알린다.

### 2.3 빈 줄의 미세 조절: "7pt 빈 줄의 마법"

단락 사이의 빈 줄은 단순한 빈 줄이 아니다. 7pt 크기의 작은 서체(charPr #88/#89, height=700)로 공백을 삽입한다.

**관찰**:
- 본문 14pt에서 빈 줄을 14pt로 넣으면 한 줄 높이 전체가 비어 너무 넓다
- 7pt(본문의 절반)로 설정하면 약 반 줄 높이의 여백이 생긴다
- 이는 paraPr의 prev/next margin 대신 사용되는 수동 여백 기법이다

**왜 이렇게 하는가**: 한컴 문서에서 paraPr의 margin 속성은 경우에 따라 예측 불가능한 동작을 한다(특히 표와 인접할 때). 7pt 빈 줄은 어떤 상황에서도 일관된 여백을 보장하는 가장 안전한 방법이다. 경험 많은 공무원들이 수년간 깨달은 노하우.

### 2.4 크기 믹싱: "14pt/12pt 교대의 위계"

한 문장 안에서 서체 크기가 바뀐다:
- "질병관리청**(14pt)**(청장 임승관)**(12pt)** 국립보건연구원**(14pt)**(원장 남재환)**(12pt)**"

**왜 이렇게 하는가**: 기관명은 14pt로 읽히게, 직함+이름은 12pt로 시각적으로 종속시킨다. 같은 크기로 쓰면 "청장 임승관"이 기관명과 동급으로 읽히는데, 2pt 차이만으로 위계를 만든다. 이것은 '직함은 기관의 부속 정보'라는 행정 문서의 철학을 반영한다.

### 2.5 표와 본문의 관계: "표 전후 여백의 비대칭"

**관찰**:
- 제목 표 아래: 빈 줄 1개(paraPr #69, 본문 시작까지 충분한 여백)
- 본문 내 표 없음(본문은 순수 텍스트)
- 담당부서 표 위: 붙임 표기 직후 배치
- 붙임 내부: 텍스트 단락 → 바로 표(캡션이 표 위에 붙음)

**데이터 표 설계의 핵심**:
- 헤더와 데이터를 구분하는 이중선(DOUBLE_SLIM 0.7mm)이 시각적 분리를 만든다
- 헤더 배경 #D9D9D9는 진하지 않은 회색으로 "정보는 있되 강하지 않은" 느낌을 준다
- 표 외곽 좌/우 테두리가 의도적으로 제거된 셀이 있어 표가 페이지 안에 자연스럽게 녹아든다

### 2.6 시각적 무게 배분

**1페이지 (이미지 1)**:
- 상단 15%: 헤더 영역 (중간 무게 -- 로고+텍스트+라인)
- 15-30%: 제목 박스 (가장 무거움 -- 26pt 볼드, 테두리 사각형)
- 30-100%: 본문 (균일한 중간 무게, 양쪽 정렬로 직사각형 텍스트 블록)
- 본문 중 빈 줄이 3번 나타나 텍스트 블록을 4개 청크로 분리

**2페이지 (이미지 2)**:
- 상단 60%: 본문 계속 (빈 줄 2개로 3개 청크)
- 60-70%: 인용문 (볼드가 시각적 무게 추가)
- 75%: 붙임 표기 (한양중고딕 bold로 시각적 전환점)
- 80-100%: 담당부서 표 (표 테두리가 시각적 앵커 역할)

**멀리서 봤을 때**: 1페이지는 "상단 짙음-중단 진행-하단 균일"의 그라데이션. 2페이지는 "상단 균일-중단 짙음(볼드+표)-하단 구조체(표)" 패턴. 전체적으로 시선이 자연스럽게 위에서 아래로 흐르며, 중간에 볼드와 표가 시선을 잡아주는 리듬.

### 2.7 "프로"가 만들었다고 느끼게 하는 세부 요소

#### 날짜 포맷
- `2026. 3. 29.(일) 12:00` -- 연, 월, 일 사이에 마침표+공백. 요일은 괄호 안에. 시간은 24시간제.
- 이는 정부 공문서 표준 날짜 표기법이다.

#### 연도 약어
- 붙임 표에서 `'26년 하반기`, `'27년 상반기` -- 백틱(`)으로 연도 축약

#### 괄호 사용의 위계
- 기관 직함: 소괄호 `(청장 임승관)`
- 영문 약어: 소괄호 `(GMP)`, `(master cell bank, MCB)`
- 사업명: 작은따옴표 `'세포기반 인공혈액 제조 및 실증 플랫폼 기술 개발'`
- 인용문: 큰따옴표 `"이번 분양이...기대한다"고 밝혔다.`
- 법률명/규칙명: 별도 따옴표 없이 풀네임 표기

#### 자간(spacing) 미세 조정
- 본문 기본: spacing 0
- 괄호 보충설명(12pt): spacing 0
- 부제목(함초롬바탕 14pt): spacing -9 ~ -4 (글자 수가 많아 줄에 맞추기 위한 미세 조정)
- 본문 밀집 구간(charPr #90/#91): spacing -7 ~ -8 (긴 문장을 한 줄 안에 넣기 위해)
- 제목(26pt): spacing -8 (큰 글씨에서 자간을 줄여 응집력 강화)

이 자간 조정은 "한 줄에 딱 맞추기" 위한 수동 미세 조정이다. 경험 있는 공무원이 출력 후 눈으로 확인하며 한 줄이 넘어가거나 어색하게 비는 것을 방지한다.

#### 위첨자(superscript) 주석 기호
- 역분화줄기세포 뒤의 `*`는 charPr #87(supscript 속성)로 위첨자 처리
- 본문 14pt와 같은 크기이나 위치가 올라가 자연스러운 각주 참조 표시

#### 하이퍼링크 스타일
- `nscb@korea.kr` -- 파란색(#0000FF) 밑줄(charPr #82: textColor="#0000FF", underline BOTTOM SOLID #0000FF)
- 붙임 내부에서도 동일 이메일이 반복될 때 같은 스타일 유지

### 2.8 XML 구현 관점의 핵심 테크닉

#### 제목 박스의 "연결 테두리" 기법
제목(행1)과 부제목(행2)은 하나의 표이지만, borderFill을 다르게 설정하여 시각적으로 하나의 박스처럼 보이게 한다:
- 행1 borderFill #6: 좌+우+**상** 실선, 하 **없음**
- 행2 borderFill #7: 좌+우+**하** 실선, 상 **없음**
- 결과: 외곽은 완전한 사각형이지만 중간 구분선은 없음. 제목과 부제목이 하나의 영역으로 읽힌다.

#### 헤더 표의 "선택적 테두리" 기법
기관명 표의 각 셀은 다른 borderFill을 사용:
- 좌측 셀(#9): 좌+상+하 있음, 우 없음
- 중앙 셀(#8): 상+하 있음, 좌우 없음
- 우측 셀(#10): 우+상+하 있음, 좌 없음
- 결과: 외곽 테두리만 표시되고, 셀 간 내부 구분선은 보이지 않음

#### 담당부서 표의 rowSpan 구조
```
| 담당 부서 (rowSpan=3) | 국립보건연구원\n난치성질환연구과 (rowSpan=3) | 책임자 | 과 장  | 김용우 | (043-249-2510) |
|                       |                                          | 담당자 | 연구관 | 이지윤 | (043-249-2533) |
|                       |                                          | 담당자 | 연구사 | 최혜영 | (043-249-2522) |
```
좌측 2개 셀이 3행 병합되어 "누가 담당하는가"를 한눈에 보여준다.

#### 데이터 표의 헤더-데이터 분리
- 헤더 행 하단: `DOUBLE_SLIM 0.7mm` (이중선) -- 헤더와 데이터를 강하게 분리
- 데이터 행 간: `SOLID 0.15mm` -- 약한 분리
- 좌측 최외곽: 테두리 없음 (`NONE`), 우측 최외곽: 테두리 없음 -- 표가 페이지에 열린 느낌

#### 붙임 헤더의 파란색 배경
- borderFill #13: 배경 `#0066FF`, 모든 변 0.12mm 실선
- 흰색 글씨(#FFFFFF) + HY헤드라인M 15pt + DROP 그림자
- 이 조합이 "공식 첨부 문서"라는 시각적 권위를 만든다

### 2.9 서체 사용의 위계 체계

| 요소 | 서체 | 크기 | 용도 |
|------|------|------|------|
| 제목 | 함초롬돋움 (fontRef=15) | 26pt bold | 최상위 시각 권위 |
| 붙임 헤더 | HY헤드라인M (fontRef=9) | 15pt | 섹션 구분 |
| 보도자료 라벨 | 함초롬바탕 (fontRef=16/19) | 14pt | 문서 유형 표시 |
| 부제목 | 함초롬바탕 | 14pt | 제목 보조 |
| 본문 | 바탕 (fontRef=4) | 14pt | 주 읽기 영역 |
| 본문 강조 | 바탕 | 14pt bold | 핵심 성과 |
| 괄호 보충 | 바탕 | 12pt | 직함, 부속 정보 |
| 주석 | 한양중고딕 (fontRef=14) | 13pt | 용어 설명 |
| 담당부서 표 | 돋움체 (fontRef=2) | 11pt | 연락처 정보 |
| 붙임 본문 | 맑은 고딕 (fontRef=3) | 13pt | 첨부 자료 |

**원칙**: 명조 계열(바탕, 함초롬바탕)은 본문과 제목에, 고딕 계열(돋움체, 맑은 고딕, 한양중고딕)은 표, 주석, 메타데이터에 사용한다. 이것이 한국 정부 문서의 전통적 서체 위계다.
