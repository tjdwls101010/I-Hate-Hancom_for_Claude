# 한국 정부 공문서 디자인 원칙

> 6건의 정부 보도자료 분석에서 추출한 디자인 원칙.
> 이 문서는 "규칙 목록"이 아닌, 각 관습이 존재하는 **이유**를 설명한다.

---

## 1. 문서 구조: 신뢰와 효율의 골격

정부 문서는 **"누가, 언제, 무엇을"** 즉시 파악할 수 있도록 설계되어 있다.
독자는 바쁜 기자, 공무원, 국민이므로 문서 전체를 읽지 않고도 핵심을 잡을 수 있어야 한다.

### 표준 구성 순서

```
┌─────────────────────────────────────┐
│  헤더 영역 (표 형식)                    │
│  ┌───────────────────────────────┐  │
│  │ 부처 로고  │  "보도자료"       │  │
│  │ 보도시점   │  배포일자         │  │
│  │ 담당부서   │  담당자/연락처     │  │
│  └───────────────────────────────┘  │
├─────────────────────────────────────┤
│  제목 (20-26pt, 굵게, 가운데 정렬)      │
│  부제목 (14-16pt, 선택적)              │
├─────────────────────────────────────┤
│                                     │
│  본문                                │
│  □ 대주제                            │
│    ㅇ 세부 설명                       │
│      - 구체적 내용                    │
│        * 부가 정보                    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │       표 (데이터 요약)         │    │
│  └─────────────────────────────┘    │
│                                     │
├─────────────────────────────────────┤
│  붙임. 첨부파일 목록                    │
│  담당부서 정보                         │
└─────────────────────────────────────┘
```

### 왜 이 순서인가

- **헤더가 표 형식인 이유**: 보도시점, 담당자 등 메타정보는 자유 텍스트가 아닌 표로 구조화한다. 이는 기자가 "이 자료를 언제 보도해도 되는가", "문의처가 어디인가"를 0.5초 안에 파악할 수 있게 하기 위함이다.
- **제목이 본문과 분리되는 이유**: 제목 영역은 시각적으로 독립된 블록이다. 독자가 문서 더미에서 훑어볼 때 제목만으로 내용을 판별할 수 있어야 한다.
- **붙임이 맨 아래인 이유**: 본문의 흐름을 깨지 않으면서, 참조 자료의 존재를 명확히 알린다.

## 2. 시각적 위계: □ ㅇ - * 체계

이 기호 체계는 한국 정부 문서의 **가장 특징적인 관습**이다. 한국 독자라면 이 기호만 보고도 "공문서"임을 즉시 인식한다.

### 기호 위계와 각 단계의 역할

| 단계 | 기호 | 용도 | 들여쓰기 |
|------|------|------|----------|
| 1단계 | □ | **대주제** 선언. 새로운 논의 영역 시작 | 없음 |
| 2단계 | ㅇ | **설명/전개**. 대주제를 풀어서 서술 | 약간 |
| 3단계 | - | **세부 항목**. 구체적 사실, 수치 | 더 들여쓰기 |
| 4단계 | * | **부가 정보**. 보충 설명, 예시 | 더 들여쓰기 |
| 특수 | ※ | **주의/참고사항**. 맥락에서 벗어나는 보충 | 문맥에 따라 |
| 특수 | ▷ | **보충 설명**. 부연이나 예시 도입 | 문맥에 따라 |

### 실제 사용 예시

```
□ 2026년 소상공인 지원 대책 주요 내용

  ㅇ 소상공인 대출 금리를 연 3.5%에서 2.0%로 인하한다.
    - 대상: 연매출 3억 원 이하 소상공인
    - 시행 시기: 2026년 4월 1일부터 적용
      * 기존 대출자는 자동 전환되며 별도 신청 불요
      * 신규 대출 한도는 업종별로 차등 적용

  ㅇ 임대료 부담 경감을 위한 세제 지원을 확대한다.
    - 임대인에게 임대료 인하분의 70%를 세액공제
      * 종전 50%에서 상향

  ※ 상세 업종별 기준은 붙임 1 참조
```

### 왜 이 체계가 효과적인가

- **번호 매기기(1, 2, 3)를 피하는 이유**: 정부 문서는 수시로 항목이 추가/삭제된다. 번호 체계는 순서 재조정이 필요하지만 기호 체계는 그렇지 않다.
- **기호가 시각적 깊이를 표현하는 이유**: □는 시각적으로 "크고 무거운" 느낌, ㅇ는 "중간", -는 "가벼운" 느낌을 준다. 기호의 시각적 무게 자체가 위계를 전달한다.
- **한 문단에 여러 단계를 혼합하지 않는 원칙**: 각 기호는 독립된 행에서 시작한다. 이는 스캔 가독성을 극대화한다.

## 3. 타이포그래피: 절제된 강조

정부 문서의 타이포그래피는 **"장식 최소화, 정보 최대화"** 원칙을 따른다.

### 서체 선택의 원리

| 요소 | 서체 | 크기 | 이유 |
|------|------|------|------|
| 제목 | HY헤드라인M, 함초롬돋움 | 20-26pt | 돋움(고딕) 계열은 제목에서 시인성이 높다 |
| 부제목 | 함초롬돋움 | 14-16pt | 제목과 같은 계열로 시각적 통일감 유지 |
| 본문 | 바탕, 함초롬바탕 | 12-14pt | 바탕(명조) 계열은 긴 텍스트의 가독성이 우수하다 |
| 표 내부 | 바탕 또는 돋움 | 10-12pt | 좁은 공간에서 정보 밀도를 높인다 |
| 주석/각주 | 바탕 | 9-10pt | 본문과 시각적으로 구분하되 읽을 수 있는 최소 크기 |

### 강조 사용의 원칙

**굵게(Bold)는 아껴 쓴다.** 정부 문서에서 Bold는 다음 경우에만 사용한다:

1. 제목과 소제목
2. 본문 내 핵심 용어 (한 문단에 1-2개 이내)
3. 표 헤더

Bold를 남용하면 강조의 효과가 사라진다. 정부 문서가 주는 "절제된 권위감"은 이 원칙에서 나온다.

### 색상은 거의 사용하지 않는다

- **본문**: 검정(#000000)만 사용. 이는 "공적 문서의 중립성"을 시각적으로 표현한다.
- **파란색(#0000FF)**: 하이퍼링크 또는 특별히 강조할 참조에만 사용
- **빨간색(#FF0000)**: 정정사항이나 경고에만 사용
- **흰색(#FFFFFF)**: 어두운 배경의 표 헤더 위 텍스트

컬러풀한 문서는 정부 공문서가 아니다. 색상의 부재 자체가 공식성의 신호다.

## 4. 표(Table) 활용: 구조화된 정보 전달

정부 문서에서 표는 장식이 아니라 **정보 압축 도구**다.

### 표가 사용되는 네 가지 상황

1. **데이터 비교**: 전년 vs 올해, 방안 A vs 방안 B
2. **조직/연락처 정보**: 담당부서, 담당자, 전화번호
3. **시간순 정보**: 추진 일정, 단계별 계획
4. **메타 정보**: 문서 헤더의 보도시점, 배포일자 등

### 표 디자인 원칙

```
┏━━━━━━━━━━━━━┳━━━━━━━━━━━━━┳━━━━━━━━━━━━━┓  ← 상단 테두리 (두껍게)
┃  항목 (헤더)  ┃  2025년      ┃  2026년      ┃  ← 배경색: 연회색 또는 연파랑
┣━━━━━━━━━━━━━╋━━━━━━━━━━━━━╋━━━━━━━━━━━━━┫  ← 헤더-본문 구분선 (두껍게)
┃  지원 금액    ┃  500억 원    ┃  800억 원    ┃  ← 본문: 흰 배경
┃  대상 인원    ┃  1만 명      ┃  1.5만 명    ┃
┗━━━━━━━━━━━━━┻━━━━━━━━━━━━━┻━━━━━━━━━━━━━┛  ← 하단 테두리 (두껍게)
```

**세부 스타일:**

- **헤더 배경**: 연회색(#D9D9D9) 또는 연파랑(#DFE6F7, #E0F4F6). 진한 색은 사용하지 않는다.
- **테두리**: 기본 0.12mm 실선. 헤더와 본문 사이는 0.7mm 이중선으로 구분하기도 한다.
- **헤더 텍스트**: 가운데 정렬, 굵게
- **데이터 텍스트**: 텍스트는 왼쪽 정렬, 숫자는 오른쪽 정렬이 일반적
- **글자 크기**: 본문보다 1-2pt 작게 (10-12pt)

### 왜 표 헤더가 연한 색인가

진한 배경색은 시선을 과도하게 끌어 본문과의 균형을 깨뜨린다. 연한 배경색은 "여기가 헤더임"을 알려주면서도 문서 전체의 차분한 톤을 유지한다.

## 5. 레이아웃: 보이지 않는 질서

### 기본 페이지 설정

| 항목 | 값 | 이유 |
|------|-----|------|
| 용지 | A4 | 한국 표준 |
| 좌우 여백 | 20mm | 인쇄 시 바인딩 공간 확보 + 시선의 여유 |
| 상하 여백 | 15mm | 정보 밀도와 여백의 균형 |
| 줄 간격 | 160% | 한글의 네모꼴 특성상 라틴 문자보다 넓은 줄 간격 필요 |
| 본문 정렬 | 양쪽 정렬 | 좌우 끝이 정돈되어 공식적 인상 부여 |
| 제목 정렬 | 가운데 정렬 | 시각적 무게중심을 페이지 중앙에 배치 |

### 줄 간격 160%의 의미

분석한 500여 개의 스타일 정의 중 408개가 160% 줄 간격을 사용했다. 이는 사실상 **정부 문서의 표준**이다. 한글은 글자 높이가 라틴 알파벳보다 크고 균일하기 때문에, 120-130% 줄 간격(영문 기준 일반적)으로는 답답해 보인다. 160%는 한글 본문에서 "읽기 편하면서도 느슨하지 않은" 최적점이다.

### 빈 문단의 역할

정부 문서에는 의도적인 빈 문단(빈 줄)이 섹션 사이에 삽입된다. 이는 단순한 여백이 아니라 **의미 단위의 시각적 구분자** 역할을 한다. 제목 아래, 표 위아래, 주제 전환 지점에 빈 문단을 넣어 독자에게 "여기서 내용이 바뀐다"는 신호를 준다.

### 양쪽 정렬(Justify)을 쓰는 이유

양쪽 정렬은 텍스트 블록의 좌우 경계를 깔끔하게 맞춰 **정돈된 인상**을 준다. 이는 공문서의 공식성을 시각적으로 강화한다. 단, 제목과 표 헤더는 가운데 정렬하여 본문과의 시각적 대비를 만든다.

## 6. 전체적인 인상: 정부 문서다움

### 정보 밀도의 원칙

정부 보도자료는 **페이지당 정보량이 많다**. 문단은 간결하고, 핵심 데이터는 표로 압축하며, 불필요한 수식어를 줄인다. "많은 내용을 적은 공간에, 그러나 읽기 쉽게"가 설계 철학이다.

이를 가능하게 하는 장치들:
- 기호 위계로 들여쓰기만으로 구조를 표현 (별도 소제목 불필요)
- 표로 비교/나열 정보를 압축
- 빈 문단으로 시각적 호흡을 확보

### 시각적 리듬

잘 만든 정부 문서를 멀리서 바라보면 다음과 같은 리듬이 보인다:

```
[헤더 표] ───── 정형화된 블록
                    ↓
[큰 제목] ───── 시각적 정점
                    ↓
[짧은 도입] ───── 1-2문장 맥락 제공
                    ↓
[□ ㅇ - * 구조] ── 본문의 핵심 (가장 긴 영역)
                    ↓
[표] ──────── 데이터 요약
                    ↓
[□ ㅇ - * 구조] ── 추가 본문 (선택적)
                    ↓
[붙임/연락처] ──── 마무리
```

이 리듬이 깨지면 — 예를 들어 표 없이 텍스트만 이어지거나, 기호 위계 없이 긴 문단이 나열되면 — 독자는 "정부 문서답지 않다"고 느낀다.

### 요약: 정부 문서 디자인의 세 가지 핵심 가치

| 가치 | 구현 방식 |
|------|----------|
| **공식성** | 색상 절제, 양쪽 정렬, 구조화된 헤더 표 |
| **효율성** | □ㅇ-* 위계, 표 활용, 높은 정보 밀도 |
| **가독성** | 160% 줄 간격, 빈 문단 구분, 명확한 서체 크기 단계 |

이 세 가치가 균형을 이룰 때 "정부 문서다운" 인상이 완성된다.
